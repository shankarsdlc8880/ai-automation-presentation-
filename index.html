Inside the brain: how it maps each attribute

The brain never matches text literally. It turns each attribute into meaning (Sea ≈ Ocean), compares it against everything it already knows, then decides — and learns from every correction. Each attribute travels through these five steps:

NormalizeClean the raw value — trim noise, fix case & formatting so it can be compared.
EmbedTurn the value into a meaning-vector (BGE-M3) so similar things sit close together.
Vector searchCompare that vector to every OTM record & past mapping to find the closest.
ScoreRank the candidates by cosine similarity and assign a confidence score.
Map / routeAbove the threshold, auto-map & store; below it, send to a human to confirm.
Now processing
VECTOR KNOWLEDGE STORE
embeddings + approved mappings · e.g., Oracle AI Vector Search / pgvector

This is not a normal table. Each record is saved as a vector — a numeric fingerprint of its meaning — so lookups match by meaning, not exact text. A new attribute finds its nearest neighbour by similarity, which is why a typo like "MARSK LINE" still resolves to Maersk. Every confirmed mapping is written back, so the store keeps getting smarter.
